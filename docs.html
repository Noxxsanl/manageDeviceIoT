📡 IoT Device Manager v1.0

Hướng dẫn cài đặt & sử dụng

Hệ thống quản lý thiết bị IoT đầy đủ: từ firmware ESP32 đọc cảm biến, truyền qua MQTT & HMAC, đến Dashboard web theo dõi thời gian thực.F

01 — Tổng quan

Hệ thống là gì?

IoT Device Manager là nền tảng quản lý & giám sát thiết bị IoT theo kiến trúc 3 tầng: Firmware phần cứng → Backend REST API → Frontend Dashboard. Toàn bộ giao tiếp giữa thiết bị và server được bảo vệ bằng HMAC-SHA256.

🌡️
Sensor Node (ESP32)
Đọc nhiệt độ & độ ẩm từ DHT22, ký HMAC và gửi lên MQTT broker mỗi 5 giây.
📶
Gateway Node (ESP32-S3)
Nhận dữ liệu từ MQTT, xác thực, ký HMAC gateway rồi forward lên Backend qua HTTP.
🖥️
Backend (Node.js)
REST API xác thực HMAC 2 lớp, lưu dữ liệu vào MySQL, ghi audit log.
🌐
Frontend (Next.js)
Dashboard quản lý thiết bị, xem biểu đồ cảm biến, audit log, quản lý user.
🛡️
Bảo mật
JWT, HMAC-SHA256, Rate limiting, RBAC 3 cấp, auto-block sau 5 lần thất bại.
📝
Audit Log
Ghi lại toàn bộ sự kiện: đăng ký thiết bị, nhận dữ liệu, auth thất bại, block thiết bị.

03 — Công nghệ

Stack công nghệ

🖥️ Backend

  • Node.js + Express + TypeScript
  • MySQL (connection pool)
  • JWT (jsonwebtoken) – 8h expiry
  • bcrypt (cost 12) - mã hoá mật khẩu
  • Helmet – HTTP security headers
  • express-rate-limit - giới hạn request
  • HMAC-SHA256 (built-in crypto)
  • Morgan – HTTP logger

🌐 Frontend

  • Next.js 15 + TypeScript
  • TailwindCSS – styling
  • Lucide React – icons
  • Axios – HTTP client
  • React Context – state management
  • App Router (Next.js)

💡 Firmware ESP32

  • PlatformIO + Arduino framework
  • DHT22 – cảm biến nhiệt độ/độ ẩm
  • PubSubClient – MQTT client
  • ArduinoJson – JSON encode/decode
  • mbedTLS – tính HMAC-SHA256
  • WiFiClientSecure + HTTPClient
  • NTP sync – đồng bộ thời gian

🏗️ Infrastructure

  • Mosquitto MQTT Broker (local LAN)
  • MySQL 8.x
  • Cổng Backend: 3000
  • Cổng Frontend: 3001
  • MQTT: 1883

🖥️ Cài đặt Backend

1
Tạo Database MySQL

Chạy script SQL để tạo database và các bảng cần thiết:

MySQL 
CREATE DATABASE iot_manager CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
USE iot_manager;

CREATE TABLE users (
  id           INT AUTO_INCREMENT PRIMARY KEY,
  username     VARCHAR(64) NOT NULL UNIQUE,
  password_hash VARCHAR(255) NOT NULL,
  role         ENUM('admin','operator','viewer') NOT NULL,
  created_at   TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  last_login   TIMESTAMP NULL
);

CREATE TABLE devices (
  id          INT AUTO_INCREMENT PRIMARY KEY,
  device_id   VARCHAR(64) NOT NULL UNIQUE,
  device_name VARCHAR(128) NOT NULL,
  device_type ENUM('sensor','gateway') NOT NULL,
  secret_key  VARCHAR(255) NOT NULL,
  status      ENUM('inactive','active','blocked') DEFAULT 'inactive',
  location    VARCHAR(256) NULL,
  fail_count  INT DEFAULT 0,
  last_seen   TIMESTAMP NULL,
  created_by  INT NULL,
  created_at  TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE sensor_data (
  id          INT AUTO_INCREMENT PRIMARY KEY,
  device_id   INT NOT NULL,
  gateway_id  INT NOT NULL,
  payload     JSON NOT NULL,
  received_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE audit_log (
  id         INT AUTO_INCREMENT PRIMARY KEY,
  event_type VARCHAR(64) NOT NULL,
  device_id  INT NULL,
  ip_address VARCHAR(64) NULL,
  user_agent TEXT NULL,
  details    JSON NULL,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

ℹ️ Phải đăng ký thiết bị qua API trước để lấy device_id và secret_key, sau đó mới điền vào config.h và flash firmware.
1
Mở dự án bằng PlatformIO

Mở VS Code → Extensions → cài PlatformIO IDE. Sau đó mở thư mục:

  • firmware/sensor-node/ — cho Sensor Node (ESP32 DOIT)
  • firmware/gateway-node/ — cho Gateway Node (ESP32-S3)
2
Đăng ký thiết bị qua API

Gọi API để lấy credentials. Thực hiện sau khi Backend đã chạy:

POST /api/devices/register (cần JWT token) 
# Đăng nhập để lấy token (qua browser hoặc curl)
curl -c cookies.txt -X POST http://localhost:3000/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username":"admin","password":"admin123"}'

# Đăng ký Gateway
curl -b cookies.txt -X POST http://localhost:3000/api/devices/register \
  -H "Content-Type: application/json" \
  -d '{"device_name":"Gateway Tầng 1","device_type":"gateway","location":"Tầng 1"}'

# Đăng ký Sensor
curl -b cookies.txt -X POST http://localhost:3000/api/devices/register \
  -H "Content-Type: application/json" \
  -d '{"device_name":"Cảm biến phòng A","device_type":"sensor","location":"Phòng A"}'
⚠️ secret_key chỉ được trả về một lần duy nhất. Hãy lưu lại ngay!

05 — Chức năng

🔐 Xác thực & Đăng nhập

Hệ thống dùng JWT cookie-based authentication. Token có hiệu lực 8 giờ.

Đăng nhập

Truy cập http://localhost:3001/login, nhập username và password. Server kiểm tra bcrypt và trả JWT cookie HttpOnly.

🔒 Rate limit: tối đa 10 lần đăng nhập / 15 phút / IP để ngăn brute-force.

Đăng xuất

Bấm avatar hoặc nút Logout ở Sidebar. Server sẽ xóa cookie JWT phía client.

Thông tin đăng nhập mặc định

Sau khi chạy npm run seed

Password: admin123

07 — Bảo mật

🛡️ Cơ chế HMAC-SHA256

Mỗi request từ thiết bị IoT phải kèm theo 2 chữ ký HMAC: một của Gateway và một của Sensor.

Cách tính HMAC

Thuật toán (C++ / ESP32) 
// Message = device_id + ":" + unix_timestamp (giây)
String message = String(DEVICE_ID) + ":" + String(unixTimestamp);

// HMAC-SHA256 với secret_key từ API
mbedtls_md_hmac(MBEDTLS_MD_SHA256, secret_key, message, output);

// Gửi dạng hex string (64 ký tự)
String hmac_hex = toHexString(output);

Kiểm tra bảo mật phía Backend

✅ Xác thực timestamp

Timestamp phải trong khoảng ±5 phút so với thời gian server. Ngăn replay attack.

✅ timingSafeEqual

So sánh HMAC bằng crypto.timingSafeEqual() để chống timing attack.

✅ Auto-block sau 5 lần thất bại

Thiết bị bị tự động chuyển sang blocked sau 5 lần auth thất bại liên tiếp.

✅ NTP Sync bắt buộc

Firmware kiểm tra NTP đã đồng bộ trước khi gửi dữ liệu. Thiếu thời gian chính xác sẽ bỏ qua gửi.

Các lớp bảo mật khác

  • 🔒 Helmet.js — HTTP security headers (XSS, clickjacking, MIME sniffing)
  • 🔒 CORS — chỉ chấp nhận request từ FRONTEND_URL
  • 🔒 JSON body limit — tối đa 10KB để ngăn DoS payload
  • 🔒 Rate limiting — Login: 10 req/15min; Device data: 60 req/min; API: 100 req/15min
  • 🔒 bcrypt cost 12 — mã hóa mật khẩu an toàn với salt
  • 🔒 JWT HttpOnly cookie — không thể đọc từ JavaScript (chống XSS)
  • 🔒 Timing-safe login — chống user enumeration qua timing attack

🎭 Phân quyền (RBAC)

Hệ thống có 3 cấp độ quyền hạn:

RoleXem thiết bịĐăng ký/BlockXóa thiết bịQuản lý user
admin ✅✅✅✅
operator ✅✅❌❌
viewer ✅❌❌❌
ℹ️ Admin tạo ra chỉ có thể tạo operator và viewer. Không thể tạo tài khoản admin khác qua API (chỉ qua npm run seed).
08 — Firmware

🌡️ Sensor Node (ESP32)

Phần cứng: ESP32 DOIT DevKit V1 + cảm biến DHT22

Chức năng

  • Kết nối WiFi tự động reconnect
  • Đồng bộ thời gian qua NTP
  • Đọc nhiệt độ + độ ẩm từ DHT22 mỗi 5s
  • Tính HMAC-SHA256 với secret_key
  • Gửi JSON lên MQTT topic local/sensors/{id}/data
  • LED nháy khi gửi thành công (GPIO 2)

Kết nối phần cứng

DHT22 DATA  → GPIO 4 (+ 10kΩ pull-up)
DHT22 VCC   → 3.3V
DHT22 GND   → GND

LED xanh     → GPIO 0 (WiFi OK)
LED đỏ       → GPIO 2 (Send OK, onboard)
⚠️ GPIO 0 là BOOT pin. Đảm bảo LED không kéo mức xuống GND lúc khởi động (dùng điện trở pull-up, không nối trực tiếp GND).

📶 Gateway Node (ESP32-S3)

Phần cứng: ESP32-S3 N16R8 (16MB Flash, 8MB PSRAM)

Chức năng

  • Subscribe MQTT topic local/sensors/+/data
  • Xác thực sensor HMAC trước khi forward
  • Ký HMAC gateway vào request
  • HTTP POST lên Backend /api/device/data
  • LED báo forward thành công (GPIO 5)
  • Bỏ qua nếu NTP chưa đồng bộ

Kết nối LED ngoài

⚠️ ESP32-S3 KHÔNG có onboard LED ở GPIO 2

LED xanh → GPIO 4 (220Ω → 3.3V) WiFi
LED vàng → GPIO 5 (220Ω → 3.3V) Forward OK

Tránh: GPIO 0 (BOOT), GPIO 19/20 (USB)

🔄 Luồng dữ liệu đầy đủ

①
Sensor đọc & ký HMAC

DHT22 trả về temperature, humidity. Sensor lấy Unix timestamp từ NTP, tính HMAC-SHA256("ESP32-SN-xxx:timestamp") với secret_key.

②
Publish lên MQTT

Sensor publish JSON đến topic local/sensors/ESP32-SN-xxx/data trên Mosquitto broker.

③
Gateway nhận & forward

Gateway subscribe wildcard, nhận message, kiểm tra sensor có trong KNOWN_SENSORS[], ký thêm HMAC gateway, rồi HTTP POST lên Backend.

④
Backend xác thực HMAC × 2

Middleware validateDevice xác thực cả gateway HMAC lẫn sensor HMAC. Nếu thất bại, tăng fail_count. Đến 5 lần → tự động block thiết bị.

⑤
Lưu database & cập nhật trạng thái

Dữ liệu lưu vào bảng sensor_data. Reset fail_count = 0, cập nhật last_seen = NOW() cho cả gateway và sensor. Ghi audit log DATA_RECV.

⑥
Dashboard hiển thị

Frontend poll dữ liệu định kỳ, hiển thị biểu đồ nhiệt độ/độ ẩm, trạng thái online/offline (dựa trên last_seen < 60s).
